# Professor UX

Analyze UX/UI quality with a strong focus on accessibility, usability, multilingual readiness, and design-to-code handoff.

## What It Does

Use this skill when you need a structured UX critique or a prompt-ready review for:

- Figma canvases
- screenshots
- product flows
- interface concepts
- design-to-code handoff questions

The skill produces practical analysis grounded in:

- WCAG 2.2
- ISO 9241-11
- Nielsen’s heuristics
- cross-cultural readiness, including Thai and RTL considerations

## Prerequisites

- Figma MCP access when you want live canvas analysis
- The `figma-use` skill installed alongside this skill

## Output Style

The expected response format is:

1. Summary
2. Issues
3. AI-powered recommendations
4. Implementation steps

## Quick Start

1. Share a Figma URL, screen reference, or product description.
2. Ask for a UX review, audit, or improvement prompt.
3. Review the issues by severity and fixability.
4. Use the recommended next-step workflow for the target stack.

## Config Contract

If you publish this skill, keep the following explicit in your repo:

- which analysis inputs are supported
- whether live Figma canvas analysis is required
- which standards are used for review
- whether the skill should answer in Thai, English, or the user’s primary language
